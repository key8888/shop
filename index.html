✧ 機能一覧 ✧

快適で安全な購買体験を実現する主要機能をご紹介します。

👜

スマートな買い物かご

セッションを利用したカート管理。購入数変更時に、DBの最新在庫と突き合わせて自動調整する安全設計です。

🛡️

アカウントロック機能

連続ログイン失敗時にアカウントを自動ロック。メモリ内でスレッドセーフに管理し、不正なアクセスを防ぎます。

🦉

多段階の注文フロー

お届け先入力 → 支払方法 → 確認 → 完了のフロー。注文確定直前の「2段階在庫チェック」で在庫超過を徹底排除します。

🔮

動的な検索・ソート

カテゴリ絞り込みに加え、「新着順」「売れ筋順」での表示切り替えを実装。サブクエリを用いた高度なJPQLを活用しています。

📜

AIチャットボット連携

ユーザーの曖昧な入力キーワードの揺らぎを吸収し、100種類以上の商品データから最適な提案を行うUIを搭載しています。

🖋️

カスタムバリデーション

@EmailCheckや@ItemCheckなど独自アノテーションを作成。DBと連携した重複チェックをスマートに実装しています。

⭐

商品レビュー機能

評価とコメントによるレビュー投稿機能を実装。購入前の比較検討をサポートします。

🚚

配送希望日時指定

注文時に配送日時を指定可能。配送スケジュール管理と利便性向上を実現しています。

🔄

ワンクリック再購入

購入履歴から商品情報を再利用し、少ない操作で再注文できる機能を提供しています。

✧ システム設計 ✧

保守性・拡張性を重視した設計を採用し、継続的な機能追加や改修に対応できるシステムを構築しています。

Post-Redirect-Get

二重送信対策 (PRGパターン)

POST処理の完了後は、必ずリダイレクト（GET）で完了画面へ遷移させ、ブラウザバックによるデータの多重登録を防止しています。

Data Transfer Object

レイヤー間の疎結合化

Web入力(Form)、DB操作(Entity)、画面表示(Bean)のクラスを明確に分離。BeanToolsで変換を集約し、不正なパラメータの混入を防ぎます。

Data Integrity

論理削除と履歴の保持

商品や会員が削除されてもレコードを消すのではなく「削除フラグ」を立てる論理削除を採用。過去の注文履歴との参照整合性を維持しています。

Dynamic Calculation

購入時単価の保持

商品の価格が変動しても過去の決済額が変わらないよう、OrderItemテーブルには注文確定時点の単価をコピーして保持する仕組みを取っています。
